# Dataset Card: Plant Disease Open Dataset

## Dataset Summary

This project uses a provenance-first dataset workflow for plant disease image classification. The public export should include only sources marked `redistributable: true` in `data/sources.yaml`.

The current filtered training layout used for the public model baseline contains:

- `73,059` images
- `117` classes
- `65,752` training images
- `7,307` validation images
- one single-image class without validation examples

The current default public export target is:

- Hugging Face dataset repo: `SSC-STUDIO/plant-disease-open-dataset`
- Local manifest: `data/sources.yaml`
- Bundle manifest: `manifest.json` generated by `tools/dataset_collector/build_dataset_bundle.py`
- Current redistributable export candidates: PlantVillage color/grayscale/segmented plus deterministic PlantVillage robustness variants
- Current full local provenance bundle: `11.177 GB`, `384,278` images; private or unclear-license sources must not be exported publicly

## Intended Uses

- Education and classroom demonstrations for computer vision in agriculture.
- Reproducible baseline training and evaluation.
- Dataset quality, label imbalance, and provenance exercises.

## Out-of-Scope Uses

- Do not use the model or dataset as the only basis for real agricultural diagnosis.
- Do not publish restricted, private, or unclear-license images.
- Do not merge sources without preserving source and license metadata.

## Source Policy

Each source must declare:

- `name`
- `url`
- `license`
- `redistributable`
- `citation`
- `local_path`
- `splits`
- `class_mapping`

Only `redistributable: true` sources are exported by `tools/dataset_collector/export_hf_dataset.py`.

## Known Limitations

- Controlled leaf datasets can overstate real-world accuracy.
- Field images vary by device, lighting, geography, crop variety, and disease stage.
- Class mappings across public datasets are not always equivalent.
- Some sources are suitable for local research but not public redistribution.

## Rebuild Commands

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
$env:PLANT_DATA_ROOT = "$PWD\.datasets"
python -c "import os; from huggingface_hub import snapshot_download; snapshot_download(repo_id='mohanty/PlantVillage', repo_type='dataset', local_dir=os.path.join(os.environ.get('PLANT_DATA_ROOT', '.datasets'), 'PlantVillage-HF'), resume_download=True)"

python tools/dataset_collector/convert_plantvillage_hf.py `
  --source-dir "$env:PLANT_DATA_ROOT\PlantVillage-HF" `
  --output-dir "$env:PLANT_DATA_ROOT\Processed\PlantVillage-Color-Classification" `
  --overwrite

python tools/dataset_collector/convert_plantvillage_hf.py `
  --config grayscale `
  --source-dir "$env:PLANT_DATA_ROOT\PlantVillage-HF" `
  --output-dir "$env:PLANT_DATA_ROOT\Processed\PlantVillage-Grayscale-Classification" `
  --overwrite

python tools/dataset_collector/convert_plantvillage_hf.py `
  --config segmented `
  --source-dir "$env:PLANT_DATA_ROOT\PlantVillage-HF" `
  --output-dir "$env:PLANT_DATA_ROOT\Processed\PlantVillage-Segmented-Classification" `
  --overwrite

python tools/dataset_collector/generate_robustness_variants.py `
  --source-dir "$env:PLANT_DATA_ROOT\Processed\PlantVillage-Color-Classification" `
  --output-dir "$env:PLANT_DATA_ROOT\Processed\PlantVillage-Robustness-Variants" `
  --target-gb 3.0 `
  --quality 98 `
  --seed 888 `
  --overwrite

python tools/dataset_collector/build_dataset_bundle.py `
  --source-manifest data/sources.yaml `
  --target-dir "$env:PLANT_DATA_ROOT\PlantDisease-Open-Bundle" `
  --no-validate-images `
  --no-hash

python tools/dataset_collector/export_hf_dataset.py `
  --bundle-dir "$env:PLANT_DATA_ROOT\PlantDisease-Open-Bundle" `
  --output-dir "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open" `
  --dataset-repo SSC-STUDIO/plant-disease-open-dataset `
  --overwrite `
  --copy-mode link

python tools/dataset_collector/filter_dataset_quality.py `
  --input-dir "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open" `
  --output-dir "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open-Filtered" `
  --metadata-csv "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open\metadata.csv" `
  --copy-mode link `
  --overwrite `
  --min-file-size 1024 `
  --min-dimension 96 `
  --min-stddev 3.0 `
  --min-entropy 1.0 `
  --near-duplicate-hamming 3
```

## Quality Filtering

Before publishing, run `tools/dataset_collector/filter_dataset_quality.py` against the Hugging Face-ready export. The filter creates a separate output directory and does not delete raw sources.

The filter rejects:

- Missing or unreadable image files.
- Files smaller than the configured byte threshold.
- Images below the configured minimum dimension.
- Very low-contrast or low-entropy images.
- Byte-identical duplicates by SHA-256.
- Optional same-label near duplicates by perceptual hash.

The filtered export includes:

- `filter_report.json`
- `rejections.csv`
- `kept_audit.csv`
- Filtered `metadata.csv`

Set `--near-duplicate-hamming 0` when deterministic augmentation variants should remain in the dataset.

## Training Layout Export

The trainer expects numeric class directories. Convert a Hugging Face-ready filtered export into `train/` and `val/` with hard links:

```powershell
python tools/dataset_collector/export_training_layout.py `
  --input-dir "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open-Filtered" `
  --output-dir "$env:PLANT_DATA_ROOT\PlantDisease-Open-Training-Filtered" `
  --metadata-csv "$env:PLANT_DATA_ROOT\HF-PlantDisease-Open-Filtered\metadata.csv" `
  --copy-mode link `
  --overwrite `
  --stratified-val-ratio 0.1 `
  --stratified-seed 888 `
  --stratified-min-val-per-class 1
```

The generated `manifest.json` records split counts and skipped validation classes.
